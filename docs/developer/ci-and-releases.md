# CI & Releases

## Continuous integration

[`.github/workflows/ci.yml`](https://github.com/mubin986/claude-code-ostad/blob/main/.github/workflows/ci.yml)
runs on pushes to `main`, on pull requests, and manually. It:

1. Installs dependencies with `npm ci`.
2. Lints.
3. Builds the app.
4. Uploads the production build output (`.next`, minus `.next/cache`) as a
   GitHub artifact named `next-build-<sha>` (retained 7 days).

## Commit linting

[`.github/workflows/commit-lint.yml`](https://github.com/mubin986/claude-code-ostad/blob/main/.github/workflows/commit-lint.yml)
enforces [Conventional Commits](conventions.md) on PRs and `main`. A
non-conforming message fails CI.

## Releases

Releases are automated by
[`.github/workflows/release-draft.yml`](https://github.com/mubin986/claude-code-ostad/blob/main/.github/workflows/release-draft.yml)
using [TriPSs/conventional-changelog-action](https://github.com/TriPSs/conventional-changelog-action):

1. Merge Conventional Commits into `main`.
2. The workflow bumps the version in `package.json`, regenerates `CHANGELOG.md`,
   commits both with `[skip ci]`, and creates a git tag.
3. It then opens a **draft** GitHub Release (via `softprops/action-gh-release`)
   with the generated notes — review and publish it manually.

!!! note "No releasable types → no release"
    If a batch of commits has no releasable types (e.g. only `docs`/`chore`), the
    changelog step is skipped and no draft release is created.

!!! warning "Don't hand-edit the changelog"
    `CHANGELOG.md` is generated by the release automation. Edit commit messages,
    not the changelog.

## Documentation

[`.github/workflows/docs.yml`](https://github.com/mubin986/claude-code-ostad/blob/main/.github/workflows/docs.yml)
builds this MkDocs site and deploys it to GitHub Pages on pushes to `main` that
touch `docs/**` or `mkdocs.yml`. See [Deployment](../deployment.md).
